博客搭建一览

刚开始接触博客要追溯到大二下学期，当时偶然的机会听说可以有一个简单的方法搭建一个属于自己的网站，可以在里面发一些自己的感想，学习心得啥啥啥的，于是就跟着网上的教程搭建起的环境。刚好这次毕设，老师给的一个选题就有关于使用Hexo+Github搭建博客的选题，阴差阳错，我又和博客结缘了。

本博客是使用Hexo+Github搭建起来的，域名和服务器使用的是Github给的免费的域名和服务器，不需要额外购买其他的服务器，hexo 可以理解为是基于node.js制作的一个博客工具，正常来说，不需要部署到我们的服务器上，我们的服务器上保存的，其实是基于在hexo通过markdown编写的文章，然后hexo帮我们生成静态的html页面，然后，将生成的html上传到我们的服务器。简而言之：hexo是个静态页面生成、上传的工具。Github差不多就是一个代码托管的工具，将这两个结合起来便可以拥有一个专属于自己的博客。

搭建过程

Node.js

上面提到，Hexo可以理解为基于node.js制作的一个博客工具，所以当你想使用Hexo之前，首先要在你的电脑上安装和搭建号Node.js的环境。下载百度进官网下载即可，安装完后记得进cmd里面要检查有没有装好

node -v

Git

电脑的Node.js环境配置好之后，由于我们的博客是要托管到Github上，所以需要一个连接我们电脑和Github的工具，这里我们需要用到的就是Git，下载安装也是直接百度进官网下载即可，下载完成后的标准就是你在文件夹下或者是桌面上鼠标右键会多两个（Git Bash/Git GUI）的选项，有的话，就说明你安装成功了

Github

由于是需要将代码托管到Github上，所以需要取Github上注册一个账号并
建立一个仓库，来用于之后的代码托管到网上，新上手的同志最好把仓库名字设置成“账户名.ghub.io”,以避免之后不必要的麻烦。

Markdown

上面提到，使用Markdown编写的文章，Hexo可以自动帮我们生成静态的html页面，所以，为了网站搭建好后可以日常在博客里面更新文章，我们还需要学习Markdown的语法和在电脑上下载书写Markdown的软件，首先软件推荐使用Markdownpad，Markdown的语法可以参考本博客里面的一篇文章（直接在搜索界面搜索Markdown即可发现）

开工

当上面四步（Node.js/Git/Github/Markdowm）都完成了，那么就可以正式的开始搭建博客了。

  • 安装Hexo

在自己认为适合的位置创个文件夹，文件夹可以直接起名为blog。

输入npm install hexo -g,开始安装Hexo

  • 检查Hexo是否安装成功

输入Hexo

  • 初始化文件夹

输入hexo init。

如果初始成功，最后一行会有“Start blogging with Hexo!”

  • 安装必备组件

输入npm install

  • Hexo初体验

先输入Hexo g，加载完后输入hexo s,当提示Hexo is running at 本地localhost网页时，从浏览器进入本地网页，如果你能看到一个大大的Hexo，说明你这一步已经成功了！

  • 设置Git的user name和email

    git config –global user.name “这里填你的Github用户名”

git config –user.email “这里填你Github的邮箱”

  • 生成密匙

输入ssh-keygen -t rsa -C “你的Github邮箱”连续按三个回车

最后的密钥存储路径在你C盘用户的.ssh文件夹里面

*添加密钥到ssh-agent

输入eval “$(ssh-agent-s)”

  • 添加生成的SSH key到ssh-agent

输入ssh-add ~/.ssh/id_rsa

  • 在Github内添加本地生成的密钥，用于Github与本地的连接

登录Github，进入头向下的setting，添加ssh，即把本地的密钥复制进去即可。

  • 查看ssh是否添加成功

    Git里输入ssh -T git@github.com

如果后面有对你的用户名说Hi，那么说明连接成功!

  • 配置_config.yml文件，修改repo值

将blog文件夹下的配置文件末尾的deploy修改为

deploy:

type: git

repository: git@github.com:Github用户名/github仓库名.git

branch: master

  • 安装扩展

npm install hexo-deployer-git –save

  • 生成以及部署

hexo d -g

生成成功后访问你的地址：http://用户名.github.io即可看到你的博客啦！
